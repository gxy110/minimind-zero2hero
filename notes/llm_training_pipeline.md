# 大模型训练标准流程

> 本文聚焦"标准流程"，帮助你建立清晰的主干认知。仓库中的 RLAIF、Agentic RL 等是主干之外的**扩展方向**，放在最后说明。

---

## 一、全局视角

```
原始文本语料
     ↓
[第1步] 预训练 (Pretrain)        → 模型学会"语言建模"
     ↓
[第2步] 监督微调 (SFT)           → 模型学会"听指令"
     ↓
[第3步] 偏好对齐 (Alignment)     → 模型学会"说人话"
     ↓
部署上线
```

这三步是业界公认的标准主干（GPT-3/4、LLaMA、Qwen 等主流模型均遵循此路径）。

---

## 二、各步骤详解

### 第 1 步：预训练 (Pretrain)

**做什么**：在海量无标注文本上训练，任务是"预测下一个 token"（自回归语言建模）。

**学到什么**：语法、事实知识、推理模式——模型变成一个"文本补全器"，但不会对话。

**数据**：网页、书籍、代码等原始文本，规模通常在 GB～TB 级别。

**本仓库对应**：`trainer/train_pretrain.py`，产出 `pretrain_*.pth`

---

### 第 2 步：监督微调 (SFT, Supervised Fine-Tuning)

**做什么**：用人工标注的"指令→回答"对继续训练，损失函数**只计算模型回答部分**（用户输入被 mask 掉）。

**学到什么**：如何按照指令格式回答问题，从"文本补全器"变成"助手"。

**数据**：问答对、对话记录，格式如：
```
[User]: 解释一下什么是梯度下降
[Assistant]: 梯度下降是一种...（这部分才计算 loss）
```

**两种变体**：
- **Full SFT**：更新全部参数，效果更好，显存需求大
- **LoRA**：冻结原始权重，只更新低秩矩阵（A、B 两个小矩阵），参数高效，是资源有限时的首选

**本仓库对应**：`trainer/train_full_sft.py` / `trainer/train_lora.py`

---

### 第 3 步：偏好对齐 (Alignment)

**做什么**：让模型的回答更符合人类期望——更有帮助、更安全、更诚实。

**为什么需要**：SFT 之后模型会回答，但不一定"好"。比如两种回答都语法正确，但一种更清晰易懂，对齐训练就是教模型区分这种差异。

**数据**：针对同一问题的"好回答 vs 差回答"对（chosen / rejected）。

#### 主流方法对比

| 方法 | 核心思路 | 是否需要奖励模型 | 难度 |
|---|---|---|---|
| **DPO** | 直接从偏好对学习，隐式优化策略 | 不需要 | 低，推荐入门 |
| **PPO** | 先训奖励模型打分，再用强化学习优化 | 需要 | 高，但效果强 |
| **GRPO** | 同一问题采样多个答案，组内相对排序 | 不需要 | 中，DeepSeek-R1 同款 |

**实践建议**：资源有限时优先用 **DPO**；追求极致效果或训推理能力时用 **PPO/GRPO**。

**本仓库对应**：`trainer/train_dpo.py` / `trainer/train_ppo.py` / `trainer/train_grpo.py`

---

### 可选：知识蒸馏 (Distillation)

**做什么**：用大模型（teacher）的输出概率分布来指导小模型（student）训练，而不只是用 0/1 的 hard label。

**目的**：让小模型学到大模型的"软知识"，在有限参数下达到更好效果。

**不是必须的**，通常用于模型压缩场景。

**本仓库对应**：`trainer/train_distillation.py`

---

## 三、扩展方向（仓库中"额外"的东西）

以下内容**不在标准三步之内**，是特定能力的专项训练：

### RLAIF（AI Feedback 替代人类 Feedback）
- 标准 RLHF 的偏好数据由人类标注，成本极高
- **RLAIF** 用另一个 AI 模型代替人类给回答打分/排序
- 本质上还是偏好对齐，只是数据来源变了
- 对应：`dataset/lm_dataset.py` 中的 `RLAIFDataset`，配合 `train_ppo.py` 使用

### Agentic RL（智能体强化学习）
- 目标：让模型学会**调用工具**（搜索、计算器、代码执行等）
- 训练方式：模型输出工具调用 → 执行工具 → 获得反馈 → 强化学习
- 这是在对齐之后叠加的**能力扩展**，不是基础流程的一部分
- 对应：`trainer/train_agent.py`

## 四、何时用哪一步——决策指南

```
你想要什么？
│
├─ 一个能理解语言的基础模型
│   └─ 只做第 1 步（Pretrain）
│
├─ 一个能对话的助手
│   └─ 第 1 步 + 第 2 步（Pretrain + SFT）
│
├─ 一个对齐良好、安全可用的助手
│   └─ 第 1 步 + 第 2 步 + 第 3 步（完整三步）
│
├─ 资源有限，想快速实验对齐
│   └─ 用已有预训练权重 + LoRA SFT + DPO
│
└─ 想让模型具备推理/思考能力
    └─ 完整三步 + 用 GRPO 在第 3 步中训练 <think> 标签
```
